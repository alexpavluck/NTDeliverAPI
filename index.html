Why the URL above routes through allorigins.win
The Metabase public endpoint does not send an Access-Control-Allow-Origin header on its responses, so browsers block direct JavaScript fetches even though the data is technically public. The Source field above wraps the real endpoint in allorigins.win, a free public CORS proxy that adds the missing header. The dashboard works as a result, but every request now passes through a third party. For ongoing or production use, replace this with a self-hosted proxy or have CORS configured correctly on the Metabase side.

About this dashboard
Originally built to test browser-direct access to the NTDeliver public Metabase endpoint. That test failed because the Metabase server does not return an Access-Control-Allow-Origin header on its GET responses, so the dashboard now routes through allorigins.win as a workaround. All filtering is performed client-side because the public API does not respect filter parameters server-side. Region is not available as a filter because the row data does not include a region column; only country is exposed.
Known limitations
Delivery Date conflates Milestone 11 (Delivery to Warehouse) and Milestone 12 (POD Received). Milestone 8 (Pre Alert / Final Docs) is not surfaced by the endpoint. Greenlight milestones have known high null rates (54-61%). Records with MDA year 2014-2018 are pre-migration and less complete.
